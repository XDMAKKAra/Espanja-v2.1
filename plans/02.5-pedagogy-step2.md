## Commit sequence

Follow `pedagogy/PLAN.md`. Rules:
- Each phase gate branches off a fresh `main`. Separate PR per gate.
- Scheduler migration runs first and is reversible — down-migration committed alongside up-migration.
- New Finnish UI copy goes to `js/ui/strings.js`. No inline strings.
- Server-authoritative grading everywhere new — nothing new inherits the `routes/progress.js:14-28` trust flaw.

## Phase gates

**Gate A — Scheduler.**
- New SR service lands as `lib/scheduler.js` (creating new `lib/` files is allowed; don't touch `lib/auth` or `lib/openai`).
- Supabase migration adds scheduler state (table or column extension per SCHEDULER.md choice).
- Existing `progress` rows backfill into the scheduler on first read (lazy migration) — no big-bang migration job.
- Unit tests for interval math, state transitions, exam-horizon clamping.
- Acceptance: a test user with 20 progress rows shows a plausible daily queue sized to their goal (Kevyt/Normaali/Intensiivinen).

**Gate B — Composer rewrite.**
- `lib/exerciseComposer.js` updated to consume scheduler output.
- Interleaving adjacency graph lands as `lib/topicGraph.js`.
- Session shape enforced: opener → 2–3 due reviews cross-topic → 1 new → cooldown.
- Fallback: if scheduler queue is empty (brand-new user), composer uses the old topic-affinity path.
- Telemetry: `session_composed` PostHog event with topic mix + due-item count.
- Acceptance: unit test proves a 15-minute session for a user with ≥30 mastered items contains ≥3 distinct topics.

**Gate C — Hints + erroneous examples.**
- `lib/hints.js` helper with per-exercise-type ladders from HINTS.md.
- Frontend "vihje" button on every exercise; ladder depth surfaced in UI.
- New exercise sub-type `grammar.subtype = "correction"` wired through generator + grader + renderer.
- 100 erroneous-example seeds added to `data/seeds/grammar.json` under the correction subtype, QA'd per Pass 2's checklist.
- Hint cache uses SHA-256 key + 24h TTL (same pattern as training-pass generators).
- Telemetry: `hint_requested` with exercise id + ladder depth.
- Acceptance: a student who answers wrong 3× on the same item sees ladder Steps 1 → 2 → 3 in order.

**Gate D — Writing rubric.**
- Rewrite the writing grader prompt in `routes/writing.js` to use the new 4-sub-dimension / 6-band rubric from RUBRIC.md.
- Output schema: `{ band, subscores: { viestinnällisyys, rakenteet, sanasto, kokonaisuus }, feedback_fi }`.
- Calibration test: `tests/writing-rubric.test.js` with 10 sample essays (marcel writes these manually, or approves AI drafts as calibration set) and expected bands. Test must pass within ±1 subscore point per essay.
- UI update: writing results screen shows the 4 subscores + qualitative feedback, not just a band letter.
- Acceptance: calibration test green; a manual eyeball on 3 new essays agrees with grader band ±1.

## Quality bar
