## Why this pass

Today the app probably picks "next exercise" by topic-affinity weights and serves seeds when available. That's serving, not teaching. The gap between serving and teaching is:

1. **Scheduling** — when should a student see an item *again*? Right now it's probably on-demand or never. Real spaced practice keyed to the exam date will move retention more than any visual polish.
2. **Interleaving** — a 15-minute session that drills only ser/estar feels productive but transfers poorly. Mixing 3–4 topics per session is harder for students but produces durable learning. The router needs to know this.
3. **Hint ladders** — when a student gets an item wrong today, what happens? Probably "incorrect, try again" or "incorrect, here's the answer." Neither teaches. A hint ladder (nudge → partial → worked example → answer) gives the student multiple chances to retrieve.
4. **Writing rubric** — `routes/writing.js` has *some* rubric driving the AI grader. Is it defensible against YTL's actual 80/65/50/35/20/10 thresholds? Probably not. A real rubric pinned to YTL bands makes Pro writing feedback worth paying for.
5. **Erroneous examples** — proven grammar-consolidation technique. New sub-type for the grammar grader.

### 4. `pedagogy/HINTS.md`
Spec for the hint ladder. Use **Adaptive Hint Sequence**:
- Per exercise type, define the ladder. Example for aukkotehtävä:
  1. Nudge: "Mitä aikamuotoa tämä lause vaatii?" (no answer)
  2. Partial: show first letter or grammatical category
  3. Worked example: show a parallel correct sentence
  4. Answer + explanation
- Ladder is triggered by repeated wrong answers OR by an explicit "vihje" button.
- Hints are pre-generated where possible (cached against `item_id`), generated on demand only for AI-composed items.
- Telemetry: track ladder depth per item — items where everyone needs Step 3 are too hard or the prompt is unclear.
- Finnish copy for every nudge.
