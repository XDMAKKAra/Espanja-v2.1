## Problem

Puheo's current palette (black + red) reads as gaming/energy-drink/horror, not education. It pattern-matches as AI slop. A Finnish high schooler's parent should glance at the site and feel "serious study tool" — not "Twitch stream."

Pass 1 shipped the token system, components, and layout primitives. The design system infrastructure is solid. This pass **only changes the brand layer** — palette, typography personality, accent usage, imagery tone — by editing the design tokens Pass 1 already established. No component rewrites. No layout changes. If you find yourself restructuring a component, stop.

## Direction

**Warm academic.** Anchor references:
- Deep navy or near-black as the primary dark (not pure `#000` — something with warmth in it, e.g. a desaturated dark teal or dark brown-navy)
- Cream / warm off-white as the primary light surface (not cold `#fff` — think paper, not screen)
- **One** warm accent: terracotta, mustard, or muted coral. Pick one, not all three. Used sparingly for CTAs, focus states, streaks, earned badges.
- Serif headlines paired with a clean sans body. The serif is the single biggest lever for killing the AI look.
- Red is reserved for "wrong answer" / error states only. It must not appear anywhere else in the brand — not in logos, not in CTAs, not in decorative accents.

Think: a restrained European study brand. Slightly Mediterranean nod is allowed (this is a Spanish-learning product) but no flag-color grabs, no castanets, no sombreros, no obvious cliché.

## Phase 1 — Brand exploration and sign-off (STOP after this)

Produce **three palette + type directions** within the warm-academic family. They should be meaningfully different from each other — not three versions of the same palette with one hue shifted. For each direction:

- 5–7 named colors with hex values: dark primary, light surface, warm accent, muted neutrals (2–3), semantic red (error-only), semantic green (success). Accessibility: every text/background pair must hit WCAG AA (4.5:1 for body, 3:1 for large text).
- Typeface pairing: one serif for headlines (name + rationale), one sans for body (name + rationale), both free/open-licensed. Must render well in Finnish (ä, ö) and Spanish (á, é, í, ó, ú, ñ, ¿, ¡) — verify the font actually supports these.
- One-sentence character statement: "This direction feels like ___ for ___."
- A small mockup of **one hero component** (the dashboard header, or the landing hero) rendered in each direction, as static HTML/CSS in a single file. No framework, no build step, use the existing Pass 1 component markup so the comparison is apples-to-apples.

Deliverable: `design/brand-explorations/` directory with three subdirs (`01-<name>/`, `02-<name>/`, `03-<name>/`), each containing a `README.md` explaining the direction and a `preview.html` rendering the mockup. Plus a top-level `design/brand-explorations/README.md` that links all three and notes the trade-offs.

6. **Do not regenerate logo or favicon in this pass** unless the current one is actively broken. Logo is its own decision — flag it as a follow-up if needed.
