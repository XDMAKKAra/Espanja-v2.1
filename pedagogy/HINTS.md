# Puheo — Hint Ladder Design

## 1. Current hint behavior

### What the app does today

**aukkotehtava** — `js/renderers/aukkotehtava.js:44`
The renderer pre-populates `#gap-fill-hint` with the seed item's `hint_fi` field unconditionally on render, before the student has attempted anything:
```js
hint.textContent = ex.hint_fi ? `Vihje: ${ex.hint_fi}` : '';
```
The hint is always visible. No button. No ladder. Example hint text: `"ammatti → ser"`. After submit, the correct answer is shown inline if wrong (`band === 'vaarin'`), and `explanation_fi` is surfaced via `onAnswer → handleSeedAnswer → #explanation-text` (`vocab.js:757–759`).

**kaannos** — `js/renderers/kaannos.js:46–50`
Same pattern: `hint_fi` rendered as a `.translate-hint` div before any attempt. Always visible.

**lauseen_muodostus** — `js/renderers/lauseenMuodostus.js:68–72`
Same pattern: `hint_fi` always visible before any attempt.

**yhdistaminen** — `js/renderers/yhdistaminen.js`
No hint field rendered at all. Wrong pairs produce a `wrong-flash` CSS animation and allow re-try, but no textual hint is ever shown.

**monivalinta (MC / legacy gap_fill / reorder / translate_mini)** — `js/screens/vocab.js`
- `gap_fill` (legacy): `hint` field rendered always-visible at line 412: `$("gap-fill-hint").textContent = \`Vihje: ${ex.hint}\``
- `reorder`: `finnishHint` always-visible at line 568
- `translate_mini`: no hint rendered at all
- MC (`renderVocabQuestion`): no hint rendered; after wrong answer the correct option is highlighted and `explanation` shown

**Scaffold system** (`vocab.js:19–64`)
There is a `_scaffoldMeta.scaffoldLevel` (0–3) that tracks how much help the student has requested at the session level via `#scaffold-help-btn`. This is a macro-level difficulty toggle, not an item-level hint ladder. Level 0 = "Ei vihjeitä — olet tulessa!" (no hints), level 1 = "Vain vihje", levels 2–3 add more scaffolding. The button increments the level per click. The adaptive-answer endpoint (`/api/adaptive-answer`) can also push `scaffoldLevel` up silently when the student is struggling (`vocab.js:118–124`).

**Summary:** There is no hint ladder today. Hints are either always-on (aukkotehtava, kaannos, lauseen_muodostus, legacy gap_fill, reorder) or absent (monivalinta, yhdistaminen, translate_mini). The "Vihje" button key exists in `strings.js` (`btn.hint`) but no renderer calls `t('btn.hint')` — the key is unused. After submission, `explanation_fi` is surfaced for seed types (Step 4 behavior exists implicitly).

## 2. Hint ladder definition (per exercise type)

Each ladder has four steps. Steps are revealed one at a time via the Vihje button (see Section 3). Step 4 is always shown automatically after submission.

---

### monivalinta (multiple choice)

**Step 1 — Nudge**
> Mieti, mitä vaihtoehtoa ympäröivä lause tai tilanne vaatii.

**Step 2 — Partial**
Eliminate one obviously wrong distractor: dim one incorrect option with 50% opacity and add the label "Tämä ei sovi tähän lauseeseen." No correct answer revealed.

**Step 3 — Worked example**
Show a parallel sentence using the same grammar rule or vocabulary item, with the answer present:
> Esimerkki: [parallel_sentence_es] → [parallel_sentence_fi_gloss]
Generated by OpenAI on demand (see Section 4).

**Step 4 — Answer + explanation**
After submit: highlight the correct option, show `explanation` field inline.
Current behavior for wrong answers in `handleAnswer` (`vocab.js:780–789`) already does this.

---

### aukkotehtava (gap-fill)

**Step 1 — Nudge**
> Ajattele, minkälainen sana tähän kohtaan sopii lauseen merkityksen kannalta.

(Today `hint_fi` is shown immediately. In the new system it becomes Step 1 content and is hidden until requested.)

**Step 2 — Partial**
Display the first letter of the answer followed by underscores for remaining characters:
> Ensimmäinen kirjain: **e _ _ _ _ _**

**Step 3 — Worked example**
A different sentence from the same topic/rule group with the gap already filled:
> Esimerkki: *El libro **está** encima de la mesa.* (sijainti → estar)
Draw from the same seed topic; can be pre-stored as `example_sentence` in seed JSON (see Section 4).

**Step 4 — Answer + explanation**
Show `correctAnswer` + `explanation_fi`. Current renderer already does this (`aukkotehtava.js:91–94`, `vocab.js:757–759`).

---

### yhdistaminen (matching)

**Step 1 — Nudge**
> Etsi pari, jonka merkitys tuntuu tutulta. Aloita siitä.

**Step 2 — Partial**
Reveal one correct pair from the set and lock both items as matched with a "Näytetty" badge. Student must complete the rest independently.

**Step 3 — Worked example**
Show a short Finnish gloss beneath each remaining unmatched Spanish item (not the Finnish partner from the game — a synonym or usage note):
> Sanahint: [spanish_word] ≈ [brief_fi_gloss]

**Step 4 — Answer + explanation**
After all pairs resolved (or timeout): reveal any unmatched pairs. Currently `yhdistaminen.js` shows "Kaikki parit yhdistetty oikein!" on completion but gives no explanation. Step 4 should add a two-line summary: number correct + one grammar/usage note from the item set.

---

### kaannos (FI→ES translation)

**Step 1 — Nudge**
> Kirjoita ensin verbi tai pääsana, sitten täydennä lauseen muut osat.

(Today `hint_fi` is shown immediately. Move to Step 1.)

**Step 2 — Partial**
Display the first word of an accepted translation as a word chip:
> Aloitussana: **[first_word]**

**Step 3 — Worked example**
A parallel Finnish sentence with its accepted Spanish translation shown in full:
> Suomi: *Hän on lääkäri.* → Espanja: *Ella es médica.*
Generated by OpenAI using the same grammar target (see Section 4).

**Step 4 — Answer + explanation**
After submit: show `bestTranslation` + `explanation` from the grader response. Already happens via `kaannos.js:114` → `handleSeedAnswer`.

---

### lauseen_muodostus (sentence construction)

**Step 1 — Nudge**
> Mieti ensin, mikä on lauseen pääverbi ja mihin kohtaan se kuuluu.

(Today `hint_fi` shown immediately. Move to Step 1.)

**Step 2 — Partial**
Reveal the required_words chips in the correct relative order (position within the sentence), without showing the full sentence:
> Sanat oikeassa järjestyksessä: [word₁] … [word₂] … [word₃]
Gaps shown as "…" for the words not in `required_words`.

**Step 3 — Worked example**
A different Spanish sentence built from a similar structure (same tense, same verb type), with Finnish gloss:
> Esimerkki: *Ella **trabaja** en Madrid todos los días.* (subjekti + verbi + paikka + aika)

**Step 4 — Answer + explanation**
After submit: show `bestTranslation` + `explanation` from grader. Already via `lauseenMuodostus.js:142` → `handleSeedAnswer`.

## 3. Trigger rules

| Trigger | Behavior |
|---|---|
| "Vihje" button click | Advance ladder by one step. Button is always rendered and always clickable before submission. |
| 2nd wrong attempt on same item | Auto-show Step 1 (insert hint text without button click; button still available to go to Step 2). Applies to aukkotehtava and kaannos, where re-attempt is possible. |
| After answer submitted | Step 4 is always shown regardless of ladder position. This is current behavior — keep it. |
| Step 4 already shown | Vihje button is hidden or disabled. |

**Button state tracking:**
- Each exercise render resets a module-level `_hintStep` counter to `0`.
- Vihje button label: `t('btn.hint')` + step indicator, e.g. "Vihje (1/3)".
- At Step 3 the button becomes "Näytä vastaus" (`t('btn.showAnswer')`) to signal that the next click reveals the answer. (Step 4 is triggered by submission, not the button — the button at Step 3 only shows the worked example; Step 4 fires on submit.)

## 4. Hint caching strategy

### Seed items (aukkotehtava, yhdistaminen, kaannos, lauseen_muodostus)

**Step 1 source:** `hint_fi` already in seed JSON. No generation needed.

**Steps 2–3 pre-generation:** Extend the seed JSON schema with two optional fields:

```jsonc
{
  "id": "aukko_ser_estar_001",
  // ... existing fields ...
  "hint_fi": "ammatti → ser",           // Step 1 — already exists
  "hint_partial": "e_",                  // Step 2 — first letter + underscores (can be computed client-side from answer[0])
  "hint_example_es": "Ella es médica en el hospital.",  // Step 3 — parallel sentence
  "hint_example_fi": "Ammatti → ser: ella es médica."   // Step 3 — gloss
}
```

`hint_partial` for aukkotehtava can be derived client-side from `answer[0]` + `answer.length - 1` underscores — no pre-storage needed. For kaannos `hint_partial` stores the first word of the canonical answer.

`hint_example_es` / `hint_example_fi` should be batch-generated offline and committed to the seed files. A one-time script calling OpenAI with: *"Give one short Spanish example sentence demonstrating [rule], different from [sentence], with Finnish gloss"* produces ~80 tokens. At 1,440 seed items this is roughly 115k tokens ≈ $0.02 at gpt-4o-mini pricing.

### AI-composed items (monivalinta from `/api/generate`)

Steps 1–2 are generated on-demand the first time a student requests them, then cached in Supabase:

**Supabase table:**
```sql
create table hints (
  exercise_id   text primary key,
  exercise_type text not null,
  step2_text    text,           -- partial hint
  step3_es      text,           -- worked example ES
  step3_fi      text,           -- worked example FI gloss
  created_at    timestamptz default now()
);
```

**Generation prompt (gpt-4o-mini, ~100 tokens output):**
```
Given this Spanish exercise:
Question: {question}
Correct answer: {correct_option}

Return JSON: {
  "step2": "One-sentence partial hint in Finnish (no answer revealed)",
  "step3_es": "A parallel correct Spanish sentence using the same rule",
  "step3_fi": "Finnish gloss of the example"
}
```

**Cache hit flow:** `GET /api/hints/:exerciseId` → returns cached rows if present, otherwise generates and stores.

**Cost estimate:**
- Hint-request rate assumption: 5% of exercise attempts reach Step 2 or Step 3
- AI-generated exercises per session: ~8–10 MC items per batch × active students
- Conservative estimate: 1,440 seed items × 5% = 72 hint generations for seeds (one-time)
- Ongoing MC: if 100 daily active students × 10 MC items × 5% hint rate = 50 generations/day × 100 tokens = 5,000 tokens/day ≈ $0.001/day. Negligible.

## 5. Telemetry

### Event schema

Log one row per hint interaction to a `hint_events` Supabase table:

```sql
create table hint_events (
  id            uuid default gen_random_uuid() primary key,
  user_id       uuid references auth.users,
  item_id       text not null,
  exercise_type text not null,          -- monivalinta | aukkotehtava | yhdistaminen | kaannos | lauseen_muodostus
  hint_step     smallint not null,      -- 1 | 2 | 3 | 4
  attempt_number smallint not null,     -- how many submits on this item before this hint
  trigger       text not null,          -- 'button' | 'auto' | 'post_submit'
  timestamp     timestamptz default now()
);
```

### Alert threshold

A nightly job (or Supabase scheduled function) queries:

```sql
select item_id, exercise_type,
       count(*) filter (where hint_step >= 3) as step3_count,
       count(*) as total_attempts,
       round(count(*) filter (where hint_step >= 3)::numeric / count(*), 2) as step3_rate
from hint_events
where timestamp > now() - interval '7 days'
group by item_id, exercise_type
having count(*) >= 20                         -- minimum sample size
   and (count(*) filter (where hint_step >= 3)::numeric / count(*)) > 0.40
order by step3_rate desc;
```

Items where more than 40% of attempts reach Step 3 within a 7-day window are flagged for review. Flagged items should be:
1. Reviewed for ambiguous wording or incorrect distractor quality (MC)
2. Checked whether `hint_fi` (Step 1) is actually helpful
3. Possibly moved to a harder CEFR band or split into an easier prerequisite item
